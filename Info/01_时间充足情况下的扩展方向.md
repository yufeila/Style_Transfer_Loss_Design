
结论：**一部分涉及，但重点变了**。

- 经典 NST：核心就是显式损失设计，`content loss + style loss + TV loss`。
- Style Injection / InstantStyle：多是**推理阶段控制**，重点在 attention、特征注入、adapter，不一定重新设计训练损失。
- StyleShot：涉及训练 style encoder，会有重建/风格一致性等训练目标，但论文重点更多是模型结构。
- LoRA / 个性化风格方法：通常涉及扩散模型原有噪声预测损失，加上身份/风格保持约束。
- QwenStyle / i2L 这类新方法更偏**微调策略、attention modulation 或 LoRA 生成**，损失设计不是唯一主线。

所以如果你想紧扣“损失函数设计”，旧 NST 更正；如果想体现前沿，可以加一章：**现代方法从“显式损失设计”转向“模型结构与推理控制”**。

---

结论：**基础版不需要大规模训练**。

经典 NST 是“直接优化生成图像像素”，不训练模型；只用预训练 VGG19 做特征提取。  
每张图跑几百到几千步优化即可，GPU 上通常可接受。  
真正需要大规模训练的是 Johnson 快速风格迁移、StyleShot、LoRA/扩散模型个性化风格这类扩展。  
所以本项目可以主线做：**无训练的损失项诊断与论文对齐 baseline**；扩展部分只做方法综述或小规模对比。  
这样更稳，也更贴合“损失函数设计”。


## baseline 的局限性

当前 baseline 的主要局限性：

1. **仍不是现代生成模型**  
    它是经典 NST，直接优化图像像素，不是 diffusion / GAN 这类会“生成新内容”的模型。
    
2. **风格强但局部噪声重**  
    Kandinsky 色块和纹理明显增强，但天空、水面、山体有较多高频纹理。
    
3. **内容结构会被局部破坏**  
    桥主体还在，但细缆线、山体边界、水面结构被风格纹理干扰。
    
4. **颜色不可控**  
    风格图颜色强烈迁移，内容图原本的天空、水面、桥体色彩被明显改变。
    
5. **空间语义不可控**  
    Gram loss 只匹配整体风格统计，不知道哪里是天空、桥、水面，所以风格纹理可能贴到不合适区域。
    
6. **每张图都要重新优化**  
    3000 step 约 5 分钟，不适合实时或批量处理。
    

这些局限正好引出后续创新实验：颜色保持、边缘保持、多风格/生成式控制。

有，但对应强度不一样。简单说：

| 方案                    | 是否有对应论文                 | 最适合引用的论文                                                                                                                                                                                                              |
| --------------------- | ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1. 自适应 $\alpha,\beta$ | 有相关论文，但不是 NST 里最经典的直接分支 | [Gatys 2015/2016] 做内容-风格权重控制；自适应权重思想可参考 [GradNorm](https://arxiv.org/abs/1711.02257)                                                                                                                                  |
| 2. 颜色保持损失             | 有，非常直接                  | [Preserving Color in Neural Artistic Style Transfer](https://arxiv.org/abs/1606.05897)                                                                                                                                |
| 3. 边缘/结构保持损失          | 有，相关性很强                 | [Deep Photo Style Transfer](https://arxiv.org/abs/1703.07511)，用可微正则约束减少结构扭曲                                                                                                                                           |
| 4. 多风格融合              | 有，非常直接                  | [Controlling Perceptual Factors in Neural Style Transfer](https://arxiv.org/abs/1611.07865)、[A Learned Representation for Artistic Style](https://arxiv.org/abs/1610.07629)、[AdaIN](https://arxiv.org/abs/1703.06868) |
| 5. 用户偏好评价             | 有，但偏评价方法，不是生成算法         | [LPIPS](https://arxiv.org/abs/1801.03924)、[ArtFID](https://arxiv.org/abs/2207.12280)、[AST-IQAD](https://arxiv.org/abs/2208.00623)                                                                                     |

我建议优先选 **方案 2 颜色保持损失** 或 **方案 3 边缘保持损失**：它们最贴近“损失函数设计”，实现成本也比多风格融合低。方案 4 展示效果可能最好，但工程量会更大。

---

[Preserving Color in Neural Artistic Style Transfer](https://arxiv.org/abs/1606.05897) 这篇论文的方法核心不是“加一个颜色损失”，而是**用颜色空间处理来保留内容图颜色**。

主要有两类方法：

1. **只迁移亮度，不迁移颜色**  
    把内容图和风格图转到亮度/色度空间，例如 YUV/YIQ。  
    只对亮度通道 $Y$ 做 NST，最后把生成图的亮度和内容图原来的色度通道合成。
    
2. **先把风格图颜色调成内容图颜色，再做 NST**  
    对风格图做线性颜色变换，让它的颜色分布接近内容图，再把这个“调色后的风格图”送进 Gatys NST。
    

所以它解决的问题是：**保留内容图颜色，同时迁移风格纹理/笔触**。  
我们前面说的 $\mathcal{L}_{color}$ 是更容易实现的近似版，不是原论文方法。


-----

它的方法可以概括成一句话：

**在 Gatys 的 Neural Style Transfer 上，加“照片真实感正则项”和“语义区域约束”，让风格迁移只改变颜色/光照风格，尽量不扭曲照片结构。**

具体有 3 个核心部分：

1. **基础仍是 NST**  
    还是用 VGG19 的内容损失和风格 Gram loss：$\mathcal{L}_{content}+\mathcal{L}_{style}$ 
2. **加入 Photorealism Regularization**  
    这是论文最关键的创新。  
    作者认为：如果输入图是照片，那么输出也应该能看作输入图经过“局部颜色变换”得到，而不是发生几何扭曲。
    
    它用 Matting Laplacian 构造正则项：$\mathcal{L}_m=\sum_{c=1}^{3}V_c[O]^T M_I V_c[O]$
    
    其中 $O$ 是输出图，$M_I$ 是由输入图 $I$ 计算出的 matting Laplacian。  
    直观理解：**惩罚那些破坏原图边缘、结构和局部颜色关系的输出。**
    
3. **加入语义分割 mask 的风格损失**  
    普通 Gram loss 会把天空风格贴到建筑上。  
    论文用语义分割，把 sky、building、water 等区域分开，只在对应语义区域内计算风格 Gram loss。
    

最终目标是：$\mathcal{L}_{total} = \sum_l \alpha_l\mathcal{L}_c^l + \Gamma\sum_l\beta_l\mathcal{L}_{s+}^l + \lambda\mathcal{L}_m$

所以它不是简单的边缘 loss，而是更强的：  
**用 Matting Laplacian 约束输出必须保持照片级结构真实感。**
